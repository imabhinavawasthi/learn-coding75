### 1Ô∏è‚É£ Understanding **async and await** in JavaScript

JavaScript is asynchronous by nature, meaning it doesn't wait for tasks like API calls or file operations to complete before moving to the next line of code. This is where **async** and **await** come into play, making asynchronous code look and behave like synchronous code, improving readability and maintainability.

### What is `async`?

The `async` keyword is used to define a function that always returns a promise. Even if you return a simple value, JavaScript automatically wraps it in a promise.

#### Example:

```javascript
async function greet() {  
   return "Hello!";
}
greet().then(console.log);  // Output: Hello!
```

Here, `greet()` returns a promise that resolves to "Hello!".
### What is `await`? ‚è∞

The `await` keyword is used inside an `async` function to pause execution until a promise is resolved. It simplifies working with promises without using `.then()`.

#### Example:

```javascript
async function fetchData() {
    let response = await fetch("https://jsonplaceholder.typicode.com/todos/1");
    let data = await response.json();
    console.log(data);
}

fetchData();
```

In this example:

- `fetch()` returns a promise that resolves with a response.
- `await` pauses execution until the response is received.
- `await response.json()` waits for the JSON data to be extracted.

### 2Ô∏è‚É£ Error Handling with `try...catch` üò£

Since `await` pauses execution, we can handle errors using `try...catch`.

#### Example:

```javascript
async function fetchData() {
    try {
        let response = await fetch("invalid-url"); // Invalid URL
        let data = await response.json();
        console.log(data);
    } catch (error) {
        console.error("Error fetching data:", error);
    }
}

fetchData();
```

Here, if an error occurs (like a network failure), the `catch` block handles it gracefully.

### Key Takeaways:

- `async` functions always return a promise.
- `await` pauses execution until a promise resolves. ‚è∞
- Error handling is easier with `try...catch`.
- It makes asynchronous code look synchronous, improving readability. üßæ
