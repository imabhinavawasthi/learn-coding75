# React `useRef` Hook

The `useRef` Hook allows you to persist values between renders.

## Key Features

- It can be used to store a **mutable value** that does **not cause a re-render** when updated.
- It can be used to access a **DOM element directly**.

## Does Not Cause Re-renders

If we tried to count how many times our application renders using the `useState` Hook, we would be caught in an **infinite loop** since this Hook itself causes a re-render.

To avoid this, we can use the `useRef` Hook instead.

---

### Example Usage

```jsx
import { useRef, useEffect } from 'react';

function RenderCounter() {
  const renderCount = useRef(1);

  useEffect(() => {
    renderCount.current += 1;
  });

  return <p>Render Count: {renderCount.current}</p>;
}
```

# Tracking Renders with `useRef`

You can use the `useRef` Hook to **track how many times a component renders** â€” without triggering re-renders on updates.

Unlike `useState`, updating a `useRef` value does **not** cause the component to re-render.

---

## Example: Count Renders Without Re-rendering

```jsx
import { useState, useEffect, useRef } from "react";
import ReactDOM from "react-dom/client";

function App() {
  const [inputValue, setInputValue] = useState("");
  const count = useRef(0);

  useEffect(() => {
    count.current = count.current + 1;
  });

  return (
    <>
      <input
        type="text"
        value={inputValue}
        onChange={(e) => setInputValue(e.target.value)}
      />
      <h1>Render Count: {count.current}</h1>
    </>
  );
}

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(<App />);
```

# Understanding `useRef()`

The `useRef()` Hook only returns **one object**.

That object has a **single property** called `current`.

---

## Initialization

When we initialize `useRef`, we provide an initial value:

```js
const count = useRef(0);
```
